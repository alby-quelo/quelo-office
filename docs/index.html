A cosa serve

  • Portare con sé un ufficio completo (documenti, browser, stampa, scanner).
  • Lavorare su più PC con la stessa chiavetta, senza toccare i dischi interni.
  • Avere file personali leggibili anche da Windows e macOS (home exFAT).
  • Salvare solo le configurazioni che interessano, senza overlay automatici opachi.

Caratteristiche principali

Sistema e desktop

  • Basata su Debian sid (live-build), locale italiano, fuso orario Europe/Rome.
  • Avvio diretto al desktop: autologin, startx, sessione LXQt con Openbox.
  • Due pannelli LXQt: versione/RAM/home/orologio in alto; menu, taskbar, rete, Bluetooth, volume, cestino, spegnimento in basso.
  • Menu personalizzato con logo Quelo; caratteri regolabili per menu e pannello inferiore.

Comportamento live

  • Ogni avvio riparte dall’immagine ISO pulita — niente persistenza Debian automatica su /usr.
  • Cache e temporanei in RAM: sessione snella.
  • Salvataggio selettivo al riavvio/spegnimento e ripristino guidato all’avvio.
  • Documenti e file personali sulla partizione QUELO-HOME (exFAT), leggibile ovunque.

Come si usa

  1. Scarica l’ISO e un pacchetto prepare-usb (completo o solo GUI) dalla pagina Release 0.71-alpha.
  2. Prepara la chiavetta dal PC host — mai dalla live bootata sulla stessa USB.

GUI grafica Linux (consigliata su Linux)

cd Quelo_prepare_usb_gui
./prepare-usb-gui.sh

Prerequisiti host: python3, python3-tk, e2fsprogs, exfatprogs, util-linux, polkit (pkexec) o sudo.

Debian/Ubuntu: sudo apt install python3 python3-tk e2fsprogs exfatprogs util-linux polkit-1

Script a terminale (CLI Linux)

cd Quelo_prepare_usb
sudo ./prepare-usb.sh

Prerequisiti host: e2fsprogs, exfatprogs, util-linux.

GUI grafica Windows (32 e 64 bit)

  1. Scarica Quelo-prepare_usb_windows.zip (oppure RAR/TAR) dalla Release.
  2. Estrai l’archivio.
  3. Doppio clic su AVVIA.bat (accetta UAC amministratore).
  4. Nella GUI: scegli ISO, chiavetta USB, dimensione persistenza; digita SI SCRIVI.

Pacchetto offline (include Python 32 bit). Requisiti: Windows 7 SP1 o superiore, privilegi amministratore.

  1. Avvia da USB (BIOS/UEFI).
  2. Allo spegnimento, usa il pulsante «Spegni» per salvare le configurazioni che vuoi conservare.

Perché uno script separato per la chiavetta?

La preparazione USB (prepare-usb.sh) va eseguita sul PC host, non dentro la live bootata sulla stessa chiavetta:

  • Sicurezza — dd e partizionamento sono distruttivi; sul host ci sono controlli e doppia conferma.
  • Affidabilità — partizionare il disco da cui si boota è tecnicamente fragile.
  • Un’ISO per tutti — stessa immagine per tutti; ognuno personalizza la propria chiavetta dopo.
